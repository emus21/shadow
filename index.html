Preamble

Before we get started, you should know the content of this website is provided for the purposes of education and are meant to be taken as an example of what Frontend Web Development may look like. Every topic discussed herein could be subjects of an entire book by themselves. Thus, this series is by no means intended to be a complete example of best practices or an example of best technology, but instead to be a long distance viewing of a very very complex topic.

HTML Basics

In this section we're going to discuss HTML and it's role in frontend web developement. HyperText Markup Language (HTML) is a language that defines the structure of your content. HTML is made up of a series of elements, which you use to enclose, or wrap, different parts of the content to make it appear a certain way, or act a certain way. The enclosing tags can make a word or image hyperlink to somewhere else, can italicize words, can make the font bigger or smaller, and so on.

Boilerplate: The Work Before The Fun

Without getting into too much detail, there are certain elements of a web page that are so commonly used that it's a good idea to have a boilerplate snippet of HTML which will allow you to get into the more interesting bits of crafting a web page without being bogged down by the some of the extremely important, yet utterly uninteresting bits of HTML structure. 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    
</body>
</html>

We're going to break this snippet down a line at a time and discuss why it's needed! 

<!DOCTYPE html>

The <!DOCTYPE> tag is a required preamble. When HTML was young (around 1991/92), doctypes were meant to act as links to a set of rules that the HTML page had to follow to be considered "good" HTML, which could mean automatic error checking and other useful things. These days, they don't do much and are basically just needed to make sure your document behaves correctly. That's all you need to know about this tag, for now. 

<html lang="en">

The <html> tag wraps all the content of the entire page and is sometimes known as the root element, in our snippet we use an attribute to hint to the browser that the content of the document is meant to be viewed by people who read and understand the English language. 

<head>

The <head> tag acts as a container for all the stuff you want to include on the HTML page that isn't content that you are showing to the page's viewers. This includes things like keywords and a page descriptions that you want to appear in search results, CSS to style your content, character set declarations, and much more. 

<meta charset="UTF-8">

The <meta> tag provides useful information that the browser can use to determine how it will behave when rendering your web page.

This sets the character set your page should use to UTF-8 which inclues most characters from the vast majority of written languages. Basically, it can now handle any textual content you might put on it. There is no reason not to set this and it can help avoid some problems later on. 

<meta http-equiv="X-UA-Compatible" content="IE=edge">

This sets a compatibility mode for legacy browsers, namely Microsoft's Internet Explorer, and tells them how they should behave when viewing the page. 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This does 2 things. First, it sets the width of the viewport to the width of the screen in the device's native resolution. This is useful to guarantee your layouts will display properly across different browsers and devices. Second, it sets the scale to start a 1.0, which means that when you initially load the page it will be zoomed to 100%. This, generally speaking, provides a better user experience while allowing the user to zoom later on, if they require it (eg: accessibility). 

<title>Document</title>

This sets the title of your page, which is the title that appears in the browser tab the page is loaded in. It is also used to describe the page when you bookmark/favorite it. 

<body>

This contains ALL the content that you want to show to web users when they visit your page, whether it's text, images, videos, games, playable audio tracks, or whatever else.
